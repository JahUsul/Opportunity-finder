# Supply-chain vet: app-store-scraper, google-play-scraper

**Date:** 2026-05-18
**Milestone:** 3 (App Store + Play Store scrapers)
**Per:** `dashboard-oss-guide.md` "Supply chain hygiene for dependencies"; OF design doc §10 "Supply chain hygiene" and §12 milestone 3 DoD
**Verdict:** ACCEPT both libraries for v0 with monitoring caveats and named fallback options.

---

## app-store-scraper

| Attribute | Value |
|---|---|
| PyPI | `app-store-scraper` |
| Repository | https://github.com/cowboy-bebug/app-store-scraper |
| License | MIT — Tier "fork-eligible" per OF design doc §5.4 |
| Maintainer | GitHub user `cowboy-bebug` |
| Status | **Archived on GitHub** (read-only; no further upstream maintenance) |
| Latest release | 0.3.5, published 2020-11-12 |
| Python compatibility | ≥3.6 |
| CVEs | None published in NVD or GitHub Advisory Database |
| Vulnerability scan | ReversingLabs Spectra Assure flagged medium-severity findings; likely transitive dependencies rather than the library itself. Run `uv pip audit` post-install to confirm. |

### Risks

- Archived repo means no upstream fix path. If Apple changes the iTunes Search API or HTML response shape, the scraper breaks and we fork or replace.
- The ~5.5-year staleness raises probability that Apple-side changes have already introduced subtle parse errors; first scrape will reveal this.

### Fallback options if the library breaks

- `app-store-web-scraper` (PyPI) — newer fork based on cowboy-bebug's work, similar API surface.
- `python-app-store-scraper` — alternative implementation, less battle-tested.
- Apple's official iTunes Search API (`https://itunes.apple.com/search`) — supported but limited to ~200 results per query and does not paginate reviews. Useful for app metadata lookup, insufficient for review ingestion alone.

## google-play-scraper

| Attribute | Value |
|---|---|
| PyPI | `google-play-scraper` |
| Repository | https://github.com/JoMingyu/google-play-scraper |
| License | MIT — Tier "fork-eligible" per OF design doc §5.4 |
| Maintainer | GitHub user `JoMingyu` (email: `mingyu.planb@gmail.com`) |
| Status | Inactive — no PyPI release in past 12+ months. Repo not archived. |
| Latest release | 1.2.7 |
| Python compatibility | ≥3.6 |
| Dependencies | Zero external dependencies — pure stdlib + requests-style HTTP |
| CVEs | None — Snyk Open Source Advisor reports no known vulnerabilities |
| Security policy | Missing |

### Risks

- Inactive but not archived — possibility of a maintenance return, but planning around it is unsafe.
- No published security policy means vulnerability disclosure path is uncertain. Mitigation: monitor the repo's open issues for security reports rather than relying on advisory channels.
- Zero-dependency design is a security positive (no transitive vuln surface) and a maintenance positive (no dependency drift).

### Fallback options if the library breaks

- `google-play-scraper-py` — alternative PyPI package with similar API surface.
- `google-play-scraper-dmi` — DMI fork with some recent maintenance activity.
- Direct HTTP scraping of Play Store pages with `httpx` + `selectolax` — viable but adds material maintenance burden.

## Verdict and acceptance criteria

Both libraries are **accepted for v0** subject to these conditions:

1. **Pin versions** in `pyproject.toml` — `app-store-scraper>=0.3.5,<0.4` and `google-play-scraper>=1.2.7,<2`. No floating `latest`.
2. **`uv pip audit` run** post-install to confirm no transitive vulnerabilities in the actual environment. Record the audit output in the milestone 3 commit message or PR.
3. **Monitor for breakage.** If scrapes return empty for two consecutive Fridays for the same store, treat as broken and escalate to fork-or-replace. The two-evenings-per-quarter risk budget in the OF risk register covers this.
4. **Re-vet quarterly** — supply-chain hygiene cadence per the OSS guide.

If either library breaks before v0 launch (July 10, 2026), the fallback plan is to switch to the named alternatives or fork the upstream and apply minimal fixes. Anticipated total time cost: 2 evenings per library per quarter.
